What is backon?

A modern evolution of backoff — a zero-dependency Python library that makes it trivial to add retry with exponential backoff to any function. Decorator, functional, and context manager APIs for both sync and async code.

⚡

Blazing Fast

~3μs fast path for simple retries. Class-based wait generators are ~15% faster than generator-based. 1.2×–3.6× faster than tenacity.

🎯

Four APIs

Decorator (@on_exception), functional (retry()), context manager (Retrying), and callable (RetryingCaller). One library, all the flexibility you need.

🔌

Zero Dependencies

Pure Python using only the standard library. No external packages required. Optional integrations for Prometheus, OpenTelemetry, structlog, and trio.

🔄

Async Native

Same API works for async def functions, sync generators, and async generators — all retried transparently.

📦

Full Type Hints

Validated with mypy, includes py.typed PEP 561 marker, modern PEP 621 packaging with PDM build system.

🧪

Test Friendly

Global disable()/enable(), context managers for testing, assertion helpers, custom sleep injection, and test_config() for full control.

Features at a Glance

  • Zero dependencies — pure Python, stdlib only
  • Four APIs — decorator, functional, context manager, callable
  • Sync and async support with transparent async detection
  • Generator support — sync and async generators retried from scratch
  • Full type hints with mypy validation and py.typed marker
  • Global toggle — backon.disable() / backon.enable()
  • Custom sleep — inject your own sleep function for testing
  • 12+ wait strategies — expo, constant, fibo, decay, runtime, and more
  • wait_combine() — sum multiple wait strategies per retry step
  • retry_with() — override retry parameters per-call on decorated functions
  • Jitter — full jitter, random jitter, or none
  • Rich callbacks — on_attempt, on_backoff, on_success, on_giveup, before_sleep
  • Circuit breaker — CLOSED/OPEN/HALF_OPEN states with automatic recovery
  • Hedging — concurrent retry requests, first-success-wins
  • Prometheus / OpenTelemetry / structlog metrics — optional, zero hard dependencies
  • Testing module — disable_retries, limit_retries, remove_backoff, assert_retried
  • Trio support — retry with trio async framework
  • Operator overloading — compose stops with | / &, waits with +
  • Iterator API — for attempt in Retrying(...)
  • Rate limiter — sliding window throttle integrated into all retry loops
  • Per-attempt timeout — via ThreadPoolExecutor / asyncio.wait_for
  • Hot loop detection — warns on 5+ retries in under 100ms
  • Context inspection — is_retrying() / get_attempt_number()
  • Dynamic backoff — return float from giveup callback for Retry-After headers

Performance

backon is faster than both tenacity and the original backoff library.

Faster than tenacity

The fast path avoids RetryState / Attempt dataclass allocations, _decide_outcome(), _call_hdlrs, and to_details() dict creation. Auto-detected via _is_fast_path() gate when no handlers, jitter, rate limiter, or attempt timeout are configured.

Quick Example

Retry on Exception

import backon

@backon.on_exception(backon.expo, ValueError, max_tries=3)
def fetch_data():
    return api.call()

Retry on Predicate

@backon.on_predicate(backon.constant, max_tries=5, interval=0.5)
def poll_status():
    return check_ready()

Functional API

result = backon.retry(
    fetch_data, backon.expo,
    exception=ValueError, max_tries=3,
)

Context Manager

with backon.Retrying(backon.expo, exception=ValueError, max_tries=3) as r:
    result = r.call(fetch_data)

Advanced Features

Enterprise-grade resilience patterns built in.

🔒

Circuit Breaker PRO

CLOSED / OPEN / HALF_OPEN states with configurable failure threshold, recovery timeout, and half-open max calls. Thread-safe.

⚡

Hedging PRO

Concurrent retry requests using ThreadPoolExecutor or asyncio tasks. First-success-wins strategy with configurable concurrency.

📊

Metrics PRO

Prometheus, OpenTelemetry, and structlog collectors. Auto-detected at import time. Tracks attempts, successes, failures, hedge requests.

⏱️

Rate Limiter

Sliding window rate limiter. Throttles calls to avoid overwhelming backends. Integrated into all retry loops.

🔎

Context Inspection

is_retrying() / get_attempt_number() via contextvars — thread-safe and async-safe.

🛠️

Trio Support

Retry with trio async framework via the backon._trio module.
