Changelog

Complete release history from the initial fork through the latest version.

Latest version: 4.2.3 (2026-07-10) · View on PyPI
4.2.3 — 2026-07-10 LATEST

Documentation: rewrite the AGENTS.md Architecture section to match the actual repository layout. The previous text pointed AI agents to backon/_sync.py, backon/_async.py, and a non-existent backon/_retry.py. The new section documents the _retry/ subpackage alongside all other modules.

4.2.2 — 2026-07-10

Fix: stop_before_delay(max_delay) actually stopping the retry loop. _decide_outcome now writes the upcoming wait into state.outcome.wait before invoking the Stop predicate, so elapsed + wait >= max_delay evaluates against the real upcoming wait rather than 0.

Documentation: Context manager section clarifies that Retrying.__exit__/__aexit__ do not re-raise the final exception. Rate Limiter section clarifies that RateLimitError is only raised by direct RateLimiter usage. Migration section adds note about __version__ requiring pip install.

4.2.1 — 2026-07-10

Replace private-module imports in README examples with the public API. Circuit Breaker, Hedging, Metrics, Testing Utilities, and auto-detection snippets now use from backon import ... for symbols already in __all__. Trio section documents backon._trio as deliberately private.

4.2.0 — 2026-07-10

Feature: RetryingCaller and AsyncRetryingCaller now accept and forward the full set of retry options. Previously they silently dropped giveup, predicate, condition, stop, the on_* handlers, retry_error_callback, raise_on_giveup, logger, before/after, and name. The Dynamic Backoff feature (giveup predicate / float return) and the Handlers API are now reachable through the callable caller surface.

Tests: 12 new tests in test_retrying_caller.py.

4.1.5 — 2026-07-10

Fix: assert_retried() / assert_not_retried() ignoring their fn argument. Both helpers now wrap the user-supplied fn in a counting spy, drive it through on_exception, and assert the spy was invoked exactly expected_tries times.

Fix: Wait generators losing pre-configured kwargs when composed with +, wait_combine(), or wait_chain(). _Wait now stores its original constructor args/kwargs and merges them with caller-supplied kwargs.

Tests: 14 new tests across test_assertions.py and test_wait_combine.py.

4.1.4 — 2026-07-10

Fix: is_retrying() / get_attempt_number() returning None inside a retry when the fast path was active. The fast paths _retry_fast_sync and _retry_fast_async now enter _retry_context_manager(state.tries) around the target() call, matching the non-fast path.

Tests: 5 new tests in test_fast_path.py.

4.1.3 — 2026-07-10

Fix: raise_on_giveup=False being silently ignored in the fast path. _retry_fast_sync and _retry_fast_async now thread the raise_on_giveup flag and return None on giveup when it is False.

Fix: Fast path raising RuntimeError("stop triggered on success") when the stop predicate fires on the same iteration where the target succeeds but the condition still asks for a retry.

4.1.2 — 2026-07-10

Fix: hedge() raising TypeError: 'NoneType' object is not callable whenever the target function raised. The hedge helpers now build a default RetryCondition via _make_default_condition(exception, giveup, predicate) before fanning out.

4.1.1 — 2026-07-09

Fix mypy type errors and ruff formatting issues in 4.1.0 release.

4.1.0 — 2026-07-09

Performance: Add fast path (_fast.py) — monolithic inline retry loop for the common case. Avoids RetryState/Attempt dataclass allocations, _decide_outcome(), _call_hdlrs, _retry_context_manager, and to_details() dict creation. ~3μs per success call (was ~22μs).

Performance: Replace generator-based wait generators with class-based _Wait subclasses. Uses direct .next(send_value) method — ~15% faster wait gen iteration.

Benchmarks: 1.2x faster than tenacity on success path (42μs vs 51μs), 3.6x faster on 3-retry path (107μs vs 387μs).

4.0.0 — 2026-07-07

Breaking: Remove dead code (_sync.py, _async.py) — 507 lines of duplicate logic replaced with DeprecationWarning stubs.

Performance: Make _config_handlers lazy — default log handlers are no longer created eagerly at decoration time. Replace time.monotonic() with time.monotonic_ns() for lower overhead.

3.8.0 — 2026-07-06

Features: wait_combine() — sum multiple wait strategies at each step. retry_with() on decorated functions — create modified copies at the call site. Generator retry support — sync and async generators automatically retried, restarting from scratch on each attempt. StructlogMetrics — structured logging via structlog. Auto-detection of instrumentation.

3.7.3 — 3.7.1 — 2026-07-05

Fix ruff formatting, _check_hot_loop thread-safety, assert_retried(), test_config()/limit_retries()/remove_backoff() no longer being no-ops, retry_unless_exception_type inheriting from retry_if_not_exception_type, Iterator API target name, _hot_loop_data thread-safety.

Refactor _retry_loop_sync/_async — extract shared decision logic into _decide_outcome, reducing ~460 lines of duplicate logic to ~140. Split _retry.py (1277 lines) into 7 focused submodules under _retry/ package. Cover 98.23% of codebase (+39 new tests).

3.7.0 — 2026-07-01

Features: RateLimiter + RateLimitError — sliding window rate limiter. attempt_timeout parameter — per-attempt timeout via ThreadPoolExecutor (sync) or asyncio.wait_for (async). Export all hidden modules to public API.

3.6.0 — 3.6.1 — 2026-06-30

Add is_retrying() / get_attempt_number() — contextvars-based retry context inspection. Dynamic backoff via giveup return value. Hot loop detection. RetryingCaller / AsyncRetryingCaller — callable objects with pre-bound exception type.

3.5.0 — 2026-06-30

Major features: CircuitBreaker + BreakerRetrying — circuit breaker pattern with CLOSED/OPEN/HALF_OPEN states. hedge() / on_hedge() / HedgingRetrying — concurrent hedging requests. MetricsCollector, PrometheusMetrics, OTelMetrics — optional Prometheus/OpenTelemetry instrumentation. Testing module — disable_retries(), limit_retries(), remove_backoff(), assertion helpers. Trio support.

3.4.0 — 2026-06-28

Add RetryCallState dataclass with elapsed, statistics, seconds_since_start, to_details(). before/after hooks. Retrying.call_state and Retrying.statistics properties.

3.3.0 — 3.2.0 — 2026-06-28

3.3.0: Performance optimization — 7.6x faster than tenacity and backoff. Inline hot functions, fast path for pre-configured handlers.

3.2.0: Operator overloading for Stop, RetryCondition, and wait generators. Iterator API. Retrying.copy(). New conditions: retry_if_not_exception_type, retry_if_not_exception_message, retry_if_exception_cause_type. New wait generators: wait_random, wait_exponential_jitter, wait_none.

3.1.0 — 2026-06-28

Add functional API retry(). Retrying context manager with .call() / .async_call(). TryAgain exception for manual retry signaling. RetryCondition / Stop type hierarchy and helper constructors. backon.disable() / backon.enable() global toggles. Version now read dynamically from importlib.metadata.

3.0.0 — 2026-06-28

Initial fork from backoff. Drop Python 3.7, 3.8, 3.9 (minimum 3.10). Use time.monotonic() instead of datetime.datetime.now() for elapsed time. Replace Poetry with PDM (PEP 621 metadata). Add py.typed marker. Modernize type hints. Add GitHub Actions CI with matrix testing (3.10–3.14).
