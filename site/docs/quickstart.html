Installation

backon requires Python 3.10+ and has zero dependencies.

pip install backon
Note: backon is a pure Python library with no external dependencies beyond the standard library. Optional integrations (Prometheus, OpenTelemetry, structlog, trio) require their respective packages.

Retry on Exception

The @on_exception decorator retries when the decorated function raises one of the specified exceptions.

import backon

@backon.on_exception(backon.expo, ValueError, max_tries=3)
def fetch_data():
    return api.call()

You can specify multiple exception types and configure all retry parameters:

@backon.on_exception(
    backon.expo, (ValueError, TimeoutError),
    max_tries=5,
    max_time=30.0,
    jitter=None,
)
def fetch():
    ...

# With a giveup callback to stop retrying for specific exceptions
@backon.on_exception(backon.expo, HTTPError, max_tries=3, giveup=lambda e: e.status_code == 404)
def fetch():
    ...

Retry on Predicate

The @on_predicate decorator retries while the predicate function returns True for the return value. By default, it retries on falsy values (operator.not_).

# Retry while the return value is None
@backon.on_predicate(
    backon.constant,
    predicate=lambda x: x is None,
    max_tries=5,
    interval=0.5,
)
def poll_status():
    return check_ready()

The predicate parameter receives the return value of the decorated function. Retry continues while the predicate returns True.

Functional API

Call a function with retry without using a decorator. The retry() function automatically detects sync vs async functions.

result = backon.retry(
    target=fetch_data,
    wait_gen=backon.expo,
    exception=ValueError,
    max_tries=3,
)

Accepts all parameters from on_exception plus condition, stop, and name:

from backon import stop_after_attempt, stop_after_delay, stop_any

result = backon.retry(
    fetch_data,
    backon.expo,
    exception=ValueError,
    stop=stop_after_attempt(5) | stop_after_delay(30.0),
)

Context Manager

The Retrying class provides a context manager interface. The with block returns the Retrying object so you can call r.call() or r.async_call().

import backon

with backon.Retrying(backon.expo, exception=ValueError, max_tries=3) as r:
    result = r.call(fetch_data)

# Access statistics after the call
print(r.statistics)
# {'start_time': ..., 'attempt_number': 2, 'idle_for': 1.5, 'elapsed': 2.3}
Note: The __exit__ / __aexit__ methods are purely a scoping device and do not re-raise the final exception. Use r.statistics after the call if you need attempt counts. The same applies to BreakerRetrying and HedgingRetrying.

Async Support

All three APIs work with async functions transparently. No special configuration required.

# Decorator
@backon.on_exception(backon.expo, ValueError, max_tries=3)
async def fetch():
    return await api.call()

# Functional
result = await backon.retry(fetch, backon.expo, exception=ValueError, max_tries=3)

# Context manager
async with backon.Retrying(backon.constant, exception=ValueError, max_tries=3, interval=0.5) as r:
    result = await r.async_call(fetch)

Callers

RetryingCaller and AsyncRetryingCaller provide a callable interface with pre-bound exception types.

# Sync caller
caller = backon.RetryingCaller(backon.expo, max_tries=3)
caller = caller.on(ValueError)
result = caller(m_function, arg1, arg2)

# Async caller
caller = backon.AsyncRetryingCaller(backon.expo, max_tries=3).on(ValueError)
result = await caller(m_async_function, arg1, arg2)

Both accept all retry configuration options: giveup, predicate, condition, stop, the on_* handlers, retry_error_callback, raise_on_giveup, logger, before/after, and name.

Generator Support

Sync and async generator functions are retried transparently. On each retry, the generator is restarted from scratch.

@backon.on_exception(backon.expo, ValueError, max_tries=3)
def gen():
    yield 1
    yield 2
    raise ValueError("fail")
    yield 3  # reached on retry

result = list(gen())  # [1, 2, 3]

@backon.on_exception(backon.expo, ValueError, max_tries=3)
async def agen():
    yield 1
    raise ValueError("fail")
    yield 2

result = [item async for item in agen()]  # [1, 2]

retry_with()

Every decorated function (sync, async, generator) exposes a .retry_with() method that returns a new decorated function with overridden parameters. The original function is unaffected.

@backon.on_exception(backon.expo, ValueError, max_tries=5)
def fetch():
    ...

# Override max_tries
wrapped = fetch.retry_with(max_tries=3)

# Override jitter
wrapped = fetch.retry_with(jitter=None)

# Override wait generator and sleep
wrapped = fetch.retry_with(wait_gen=backon.constant, interval=0.1, sleep=lambda s: None)

Custom Sleep

Replace the default sleep function for testing or special environments.

@backon.on_exception(
    backon.expo, ValueError, max_tries=3,
    sleep=lambda s: print(f"waiting {s}s"),
)
def f():
    ...

# With asyncio.Event for testing
import asyncio
event = asyncio.Event()
@backon.on_exception(
    backon.expo, ValueError, max_tries=3,
    sleep=backon.sleep_using_event(event),
)
async def f():
    ...

Global Toggle

Useful in tests to disable retry logic globally. When disabled, the decorated function is called directly without any retry.

backon.disable()   # skip retry, call function directly
backon.enable()    # re-enable retry

Per-instance toggle via Retrying.enabled:

r = backon.Retrying(backon.expo, exception=ValueError, max_tries=3)
r.enabled = False
result = r.call(fn)  # no retry
